## 1단계 프롬프트 — 프로젝트 구조 · 문제 데이터 · 기초 UI

```
아래 PRD를 참고하여 상식 퀴즈 게임의 프로젝트 기초를 구현해줘.

[구현 범위]
1. 폴더/파일 구조 생성
   - index.html (시작 화면)
   - style.css (전체 스타일)
   - js/app.js (메인 진입점)
   - js/data.js (문제 데이터 로드)
   - data/questions.json (문제 데이터)

2. 문제 데이터(questions.json) 작성
   - 카테고리 4개: 한국사, 과학, 지리, 일반상식
   - 카테고리별 난이도(초급/중급/고급) 각 3~4문제씩, 총 40문제 이상
   - 각 문제 구조: id, category, difficulty, question, options(4개), answer(index), hint, explanation
   - 보기 순서는 나중에 랜덤 셔플 예정이므로 원본 데이터는 고정 순서로 저장

3. 시작 화면(index.html + style.css) 구현
   - 게임 타이틀 표시
   - 닉네임 입력 필드 (빈값 제출 방지)
   - 카테고리 선택 버튼 5개: 한국사, 과학, 지리, 일반상식, 전체
   - 리더보드 보기 버튼
   - 성인 타겟 UI: 깔끔하고 가독성 높은 디자인

4. 난이도 선택 화면 구현
   - 선택한 카테고리명 표시
   - 난이도 버튼 3개: 초급(+5점), 중급(+10점), 고급(+15점)
   - 난이도별 기본 점수 안내 텍스트 포함

[기술 스택]
- HTML / CSS / JavaScript (순수 바닐라, 프레임워크 없음)
- 데이터 저장: LocalStorage
- 문제 데이터: questions.json (fetch로 로드)

[주의사항]
- 모든 텍스트는 한국어
- 화면 전환은 단일 HTML에서 섹션 show/hide 방식으로 구현
- 웹 브라우저 데스크탑 환경 기준
```

### 1단계 검증 체크리스트 (PRD 연결)

#### 파일 구조
- [ ] `index.html`, `style.css`, `js/app.js`, `js/data.js`, `data/questions.json` 파일이 모두 생성되었는가 `(PRD §7 기술 스택)`
- [ ] 단일 HTML 파일 내 섹션 show/hide 방식으로 화면 전환이 구성되었는가 `(PRD §6 비기능)`

#### 문제 데이터
- [ ] 카테고리가 정확히 4개(한국사, 과학, 지리, 일반상식)인가 `(PRD §4.1)`
- [ ] 총 문제 수가 40문제 이상인가 `(PRD §4.1)`
- [ ] 모든 문제에 `id` 필드가 있으며 카테고리별 고유 식별자인가 `(PRD §8)`
- [ ] 모든 문제에 `difficulty` 필드가 초급/중급/고급 중 하나인가 `(PRD §4.1, §8)`
- [ ] 모든 문제에 `options` 배열이 정확히 4개인가 `(PRD §4.1)`
- [ ] 모든 문제에 `answer` 필드가 0~3 사이의 인덱스 값인가 `(PRD §8)`
- [ ] 모든 문제에 `hint` 필드가 존재하고 비어있지 않은가 `(PRD §8)`
- [ ] 모든 문제에 `explanation` 필드가 존재하고 비어있지 않은가 `(PRD §8)`

#### 시작 화면
- [ ] 게임 타이틀이 화면에 표시되는가 `(PRD §5.1)`
- [ ] 닉네임 입력 필드가 있으며 빈값 제출 시 경고가 뜨는가 `(PRD §5.1)`
- [ ] 카테고리 버튼이 5개(한국사/과학/지리/일반상식/전체) 모두 렌더링되는가 `(PRD §5.1)`
- [ ] 리더보드 보기 버튼이 존재하는가 `(PRD §5.1)`
- [ ] 카테고리 버튼 클릭 시 난이도 선택 화면으로 전환되는가 `(PRD §3 사용자 흐름)`

#### 난이도 선택 화면
- [ ] 선택한 카테고리명이 화면에 표시되는가 `(PRD §5.2)`
- [ ] 난이도 버튼이 초급/중급/고급 3개 모두 있는가 `(PRD §5.2)`
- [ ] 각 난이도 버튼에 기본 점수 안내(초급 5점/중급 10점/고급 15점)가 표시되는가 `(PRD §5.2, §4.4)`
- [ ] 난이도 선택 후 퀴즈 화면으로 전환되는가 `(PRD §3 사용자 흐름)`
- [ ] 이전 화면(시작 화면)으로 돌아가는 뒤로가기 버튼이 있는가 `(UX 기본)`
- [ ] 전체 UI가 한국어로만 표시되는가 `(PRD §6 비기능)`

## 2단계 프롬프트 — 핵심 게임 로직

```
1단계에서 만든 프로젝트 구조를 바탕으로 핵심 게임 로직을 구현해줘.

[구현 범위]
1. 문제 출제 로직 (js/quiz.js 신규 생성)
   - 선택한 카테고리 + 난이도에 맞는 문제만 필터링
   - 문제 순서 랜덤 셔플 (Fisher-Yates 알고리즘)
   - 보기(options) 순서 랜덤 셔플 (정답 인덱스 재계산 포함)
   - 문제를 1개씩 순차 표시

2. 퀴즈 화면 구현
   - 진행 상황 표시 (예: 3/10)
   - 현재 카테고리 / 난이도 / 누적 점수 표시
   - 문제 텍스트 표시
   - 4개 보기 버튼 (클릭 후 즉시 비활성화)
   - 힌트 버튼 (문제당 1회 제한, 사용 여부 상태 관리)

3. 즉시 피드백 구현
   - 정답 선택 시: 해당 버튼 초록색 하이라이트 + "정답입니다!" 메시지
   - 오답 선택 시: 선택 버튼 빨간색 + 정답 버튼 초록색 표시 + explanation 해설 텍스트
   - 다음 문제 버튼 표시

4. 점수 계산 로직 (js/score.js 신규 생성)
   - 난이도별 기본 점수: 초급 +5점, 중급 +10점, 고급 +15점
   - 빠른 정답 보너스: 7초 이내 정답 시 +5점 추가
   - 힌트 사용 후 정답: 해당 문제 기본 점수 × 50% (소수점 버림)
   - 오답: 0점
   - 누적 점수 실시간 업데이트

5. 키보드 단축키
   - 숫자키 1~4로 보기 선택 가능
   - 보기 활성화 상태일 때만 동작

[주의사항]
- 보기 셔플 시 정답 인덱스가 정확히 재계산되어야 함
- 힌트 사용 여부는 문제별로 독립적으로 관리
- 점수 계산은 score.js에서 순수 함수로 구현 (상태 비의존)
```

### 2단계 검증 체크리스트 (PRD 연결)

#### 문제 필터링 및 셔플
- [ ] 선택한 카테고리에 해당하는 문제만 출제되는가 `(PRD §4.1)`
- [ ] "전체" 선택 시 4개 카테고리 문제가 모두 포함되는가 `(PRD §4.1)`
- [ ] 선택한 난이도에 해당하는 문제만 출제되는가 `(PRD §4.1)`
- [ ] 문제 순서가 매 게임마다 다르게 랜덤 셔플되는가 `(PRD §4.1)`
- [ ] 보기 순서가 랜덤 셔플되는가 `(PRD §4.1)`
- [ ] 보기 셔플 후 정답 인덱스가 올바르게 재계산되는가 `(PRD §4.1)`

#### 퀴즈 화면
- [ ] 진행 상황이 "현재문제/전체문제" 형식으로 표시되는가 `(PRD §5.3)`
- [ ] 카테고리, 난이도, 누적 점수가 화면에 표시되는가 `(PRD §5.3)`
- [ ] 문제 텍스트가 정확히 표시되는가 `(PRD §5.3)`
- [ ] 보기 버튼이 4개 표시되며 클릭 후 즉시 비활성화되는가 `(PRD §4.2)`
- [ ] 힌트 버튼이 존재하며 문제당 1회만 사용 가능한가 `(PRD §4.2)`
- [ ] 힌트 버튼 클릭 시 hint 텍스트가 표시되는가 `(PRD §8)`
- [ ] 힌트 사용 후 버튼이 비활성화(재사용 불가)되는가 `(PRD §4.2)`

#### 즉시 피드백
- [ ] 정답 선택 시 해당 버튼이 초록색으로 하이라이트되는가 `(PRD §4.3)`
- [ ] 정답 선택 시 "정답입니다!" 메시지가 표시되는가 `(PRD §4.3)`
- [ ] 오답 선택 시 선택 버튼이 빨간색으로 표시되는가 `(PRD §4.3)`
- [ ] 오답 선택 시 정답 버튼이 초록색으로 표시되는가 `(PRD §4.3)`
- [ ] 오답 선택 시 explanation 해설 텍스트가 표시되는가 `(PRD §4.3)`
- [ ] 다음 문제 버튼이 피드백 후 표시되는가 `(PRD §4.3)`

#### 점수 계산
- [ ] 초급 정답 시 +5점이 적용되는가 `(PRD §4.4)`
- [ ] 중급 정답 시 +10점이 적용되는가 `(PRD §4.4)`
- [ ] 고급 정답 시 +15점이 적용되는가 `(PRD §4.4)`
- [ ] 7초 이내 정답 시 +5점 보너스가 추가되는가 `(PRD §4.4)`
- [ ] 힌트 사용 후 정답 시 기본 점수의 50%만 부여되는가 `(PRD §4.4)`
- [ ] 오답 시 0점이며 점수 변동이 없는가 `(PRD §4.4)`
- [ ] 누적 점수가 퀴즈 화면에서 실시간으로 업데이트되는가 `(PRD §5.3)`

#### 키보드 단축키
- [ ] 1~4번 키로 보기 선택이 동작하는가 `(PRD §6 비기능)`
- [ ] 보기가 비활성화된 상태(피드백 중)에서는 키 입력이 무시되는가 `(PRD §6 비기능)`

## 3단계 프롬프트 — 타이머 · 힌트 · 리더보드 · 결과 화면

```
2단계까지 구현된 퀴즈 게임에 고급 기능을 추가해줘.

[구현 범위]
1. 타이머 기능 (js/timer.js 신규 생성)
   - 문제당 20초 카운트다운
   - 퀴즈 화면에 타이머 바(progress bar) 시각적 표시
   - 타이머 바: 20초 → 0초로 줄어드는 애니메이션 (색상 변화: 초록 → 노랑 → 빨강)
   - 타임오버 시: 오답 처리 + "시간 초과!" 메시지 + 정답 표시
   - 빠른 정답 보너스 계산을 위한 경과 시간 측정 (7초 이내 여부)
   - 피드백 화면 표시 중 타이머 정지

2. 힌트 텍스트 표시 UI 완성
   - 힌트 버튼 클릭 시 hint 텍스트를 문제 아래에 표시
   - 힌트 사용 시 "이 문제는 힌트를 사용했습니다 (점수 50%)" 안내 표시
   - 힌트 사용 여부를 시각적으로 구분 (버튼 스타일 변경)

3. 결과 화면 구현
   - 최종 점수 크게 표시
   - 만점 대비 비율 표시 (예: 75점 / 100점 = 75%)
   - 카테고리별 정답 수 / 전체 문제 수 표시
   - 리더보드 내 순위 표시
   - 버튼 3개: 재도전(같은 설정으로 재시작), 카테고리 변경(시작 화면으로), 종료(탭 닫기 또는 초기화)

4. 리더보드 기능 (js/leaderboard.js 신규 생성)
   - LocalStorage에 닉네임 + 점수 + 날짜 + 카테고리 + 난이도 저장
   - 전체 순위: 점수 기준 내림차순 상위 10명
   - 카테고리별 순위: 해당 카테고리 점수 기준 상위 10명
   - 동점자 처리: 날짜 기준 최신 기록 우선
   - 리더보드 화면: 전체/카테고리별 탭 전환

[주의사항]
- 타이머는 문제가 넘어갈 때마다 반드시 초기화 및 재시작
- 게임 종료(탭 닫기 등) 전 리더보드 저장이 완료되어야 함
- 결과 화면의 "재도전" 버튼은 동일 닉네임/카테고리/난이도 유지
- 화면 전환 속도 0.3초 이내 유지 (PRD §6)
```

### 3단계 검증 체크리스트 (PRD 연결)

#### 타이머
- [ ] 매 문제 시작 시 타이머가 20초로 초기화되는가 `(PRD §4.2)`
- [ ] 타이머 바가 퀴즈 화면에 시각적으로 표시되는가 `(PRD §5.3)`
- [ ] 타이머 바가 시간 경과에 따라 줄어드는 애니메이션이 동작하는가 `(PRD §5.3)`
- [ ] 타이머 색상이 시간에 따라 변하는가 (초록→노랑→빨강) `(UX)`
- [ ] 타임오버 시 오답으로 처리되는가 `(PRD §4.2)`
- [ ] 타임오버 시 "시간 초과!" 메시지와 정답이 표시되는가 `(PRD §4.2, §4.3)`
- [ ] 보기 선택 후 피드백 화면에서 타이머가 정지하는가 `(PRD §4.2)`
- [ ] 빠른 정답 보너스(7초 이내)가 타이머 기준으로 정확히 계산되는가 `(PRD §4.4)`
- [ ] 다음 문제로 넘어갈 때 타이머가 정확히 재시작되는가 `(PRD §4.2)`

#### 힌트
- [ ] 힌트 버튼 클릭 시 hint 텍스트가 문제 아래에 표시되는가 `(PRD §4.2, §8)`
- [ ] 힌트 사용 시 "점수 50%" 안내 메시지가 표시되는가 `(PRD §4.2)`
- [ ] 힌트 버튼이 사용 후 시각적으로 변경(비활성 스타일)되는가 `(PRD §4.2)`
- [ ] 힌트 사용 후 정답 시 실제로 50% 점수만 부여되는가 `(PRD §4.4)`

#### 결과 화면
- [ ] 최종 점수가 크게 표시되는가 `(PRD §5.5)`
- [ ] 만점 대비 득점 비율이 표시되는가 `(PRD §5.5)`
- [ ] 카테고리별 정답 수가 표시되는가 `(PRD §5.5)`
