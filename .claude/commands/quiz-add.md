# 퀴즈 추가

`data/questions.json`에 새 문제를 1개 추가해 줘.

- **카테고리:** $1
- **난이도:** $2

---

## Step 1 — 현재 상태 파악

`data/questions.json`을 읽고 아래를 확인해:

1. **다음 ID 결정** — 카테고리별 접두사 규칙:
   - 한국사 → `KH`, 과학 → `SC`, 지리 → `GE`, 일반상식 → `GK`
   - 해당 카테고리의 마지막 번호 + 1로 3자리 패딩 (예: `GK-012`)

2. **정답 분포 집계** — 전체 문제의 정답 인덱스(0~3)별 개수를 세어,
   가장 적게 사용된 인덱스를 새 문제의 정답 위치로 우선 배치할 것.

3. **난이도 분포 집계** — $2 난이도가 현재 과다하면(60% 초과) 경고 후 계속 진행.

---

## Step 2 — 문제 생성

아래 필드를 모두 채워 문제를 작성해:

```json
{
  "id": "...",
  "category": "$1",
  "difficulty": "$2",
  "question": "...",
  "options": ["...", "...", "...", "..."],
  "answer": 0,
  "hint": "...",
  "explanation": "..."
}
```

**작성 규칙:**
- `options`는 반드시 4개
- `answer`는 Step 1에서 선택한 최소 빈도 인덱스에 정답을 배치
- `hint`는 정답을 직접 노출하지 않되 방향을 제시
- `explanation`은 정답 근거와 추가 맥락을 포함 (2~3문장)

---

## Step 3 — 검증 (추가 전 필수)

아래 항목을 순서대로 점검하고, 결과를 표로 출력해:

| 검증 항목 | 기준 | 결과 |
|---|---|---|
| ID 중복 없음 | 기존 ID와 겹치지 않음 | ✅ / ❌ |
| 필드 완전성 | 7개 필드 모두 존재 | ✅ / ❌ |
| 보기 개수 | options 정확히 4개 | ✅ / ❌ |
| 정답 인덱스 범위 | answer가 0~3 사이 | ✅ / ❌ |
| 최상급 표현 검사 | question·options에 가장/최초/최대/최소/최고/최장/최단/최강/유일/처음/제일 포함 시 → 측정 기준 명시 여부 확인 | ✅ 기준 명시 / ⚠️ 기준 미명시 / ➖ 해당 없음 |
| 정답 편중 경고 | 추가 후 특정 인덱스가 40% 초과 시 경고 | ✅ / ⚠️ |
| 난이도 편중 경고 | 추가 후 특정 난이도가 60% 초과 시 경고 | ✅ / ⚠️ |

❌ 항목이 있으면 수정 후 재검증. ⚠️ 항목은 내용을 명시하고 진행.

---

## Step 4 — 파일 업데이트

검증 통과 후 `data/questions.json`의 배열 마지막에 새 문제를 추가해.

완료 후 아래를 출력해:
- 추가된 문제 전체 JSON
- 업데이트 후 전체 문제 수
- 업데이트 후 정답 분포 요약
