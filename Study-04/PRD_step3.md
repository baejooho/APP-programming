# PRD Step 3 — 사용자 프로필 & 레시피 저장

## 개요
사용자 프로필을 생성하고 Step 2에서 마음에 든 레시피를 저장·관리할 수 있는 기능을 제공한다.

## 목표
- 로그인 없이도 기본 사용이 가능하되, 프로필 생성 시 데이터가 영구 보존된다.
- 저장된 레시피를 나중에 다시 꺼내보고 재활용할 수 있어야 한다.

## 기능 요구사항

### FR-1. 사용자 프로필 생성
- 닉네임, 프로필 이미지(선택) 입력으로 간단히 생성한다.
- 별도 이메일/비밀번호 없이 브라우저 로컬 스토리지 기반으로 동작한다 (MVP 범위).
- 프로필 정보:
  ```json
  {
    "id": "uuid",
    "nickname": "홍길동",
    "avatar_url": "",
    "created_at": "2026-04-07T00:00:00Z",
    "preferences": {
      "servings": 2,
      "max_cooking_time": 30,
      "diet": []
    }
  }
  ```
- 선호 인원수, 조리 시간, 식단 제한을 프로필에 저장하여 Step 2 기본값으로 자동 적용한다.

### FR-2. 레시피 저장
- Step 2의 "저장하기" 버튼 클릭 시 해당 레시피를 프로필에 저장한다.
- 저장 데이터:
  ```json
  {
    "id": "uuid",
    "saved_at": "2026-04-07T00:00:00Z",
    "recipe": { /* Step 2 레시피 JSON */ },
    "source_ingredients": ["계란", "당근"],
    "memo": ""
  }
  ```
- 동일 레시피 중복 저장 방지 (요리명 기준).

### FR-3. 저장된 레시피 목록 (마이 레시피)
- 저장된 레시피를 최신 순으로 카드 목록으로 표시한다.
- 각 카드에서:
  - 레시피 상세 보기 (조리법 전체)
  - 메모 추가/수정
  - 삭제
- 요리명 또는 재료로 검색 가능.

### FR-4. 프로필 페이지
- 저장 레시피 수, 가장 많이 사용한 재료 Top 3를 통계로 표시한다.
- 선호 설정 변경 가능.

## 비기능 요구사항
- 모든 데이터는 `localStorage`에 JSON으로 저장한다 (MVP).
- 향후 확장을 위해 저장/불러오기 로직을 별도 `storage.js` 모듈로 분리한다.
- 데이터 구조는 추후 서버 DB 연동을 고려하여 설계한다.

## 데이터 구조 (localStorage)
```
fridge_app_profile   → 사용자 프로필 객체
fridge_app_recipes   → 저장된 레시피 배열
```
