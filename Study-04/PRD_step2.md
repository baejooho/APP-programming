# PRD Step 2 — 레시피 생성

## 개요
Step 1에서 추출한 재료 목록을 `qwen/qwen3.6-plus:free` 모델에 전달하여 맞춤 레시피를 생성한다.

## 목표
- 보유 재료 기반으로 실제로 만들 수 있는 레시피를 추천한다.
- 레시피는 구조화된 형태로 제공하여 사용자가 바로 따라 할 수 있어야 한다.

## 기능 요구사항

### FR-1. 레시피 생성 API 호출
- 사용 모델: `qwen/qwen3.6-plus:free`
- 입력: Step 1의 재료 배열 + 사용자 옵션(인원수, 조리 시간, 식단 제한)
- 시스템 프롬프트:
  ```
  당신은 요리 전문가입니다.
  주어진 재료로 만들 수 있는 레시피 3가지를 추천해주세요.
  반드시 아래 JSON 형식으로만 응답하세요.
  {
    "recipes": [
      {
        "name": "요리명",
        "description": "한 줄 설명",
        "cooking_time": "조리 시간(분)",
        "servings": "인원",
        "difficulty": "쉬움|보통|어려움",
        "ingredients": [{"name": "재료명", "amount": "양"}],
        "steps": ["1. ...", "2. ..."],
        "missing_ingredients": ["없으면 아쉬운 재료"]
      }
    ]
  }
  ```

### FR-2. 사용자 옵션 입력
- 인원수: 1~8명 (기본값: 2명)
- 최대 조리 시간: 15 / 30 / 60분 이상 선택
- 식단 제한: 없음 / 채식 / 글루텐 프리 (체크박스)

### FR-3. 레시피 카드 표시
- 생성된 레시피 3개를 카드 형태로 나란히 표시한다.
- 각 카드에 포함되는 정보:
  - 요리명, 한 줄 설명
  - 조리 시간, 인원, 난이도 뱃지
  - 사용 재료 목록 (보유 재료는 녹색, 부족한 재료는 회색으로 구분)
  - 조리 순서 (단계별 펼쳐보기)
- "저장하기" 버튼: Step 3(사용자 프로필)에 레시피를 저장한다.
- "다시 생성" 버튼: 동일 재료로 새 레시피를 재요청한다.

## 비기능 요구사항
- 스트리밍 응답(`stream: true`)을 사용하여 레시피가 생성되는 과정을 실시간으로 표시한다.
- JSON 파싱 실패 시 원문 텍스트를 그대로 표시하고 재시도 버튼을 제공한다.
