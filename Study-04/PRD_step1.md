# PRD Step 1 — 냉장고 이미지 인식

## 개요
사용자가 냉장고 사진을 업로드하면 `google/gemma-3-27b-it:free` 모델이 이미지를 분석하여 식재료 목록을 추출한다.

## 목표
- 사용자가 별도의 입력 없이 사진 한 장만으로 재료 목록을 얻을 수 있어야 한다.
- 추출된 재료 목록은 Step 2(레시피 생성)의 입력으로 그대로 사용된다.

## 기능 요구사항

### FR-1. 이미지 업로드
- 사용자는 파일 선택 또는 드래그&드롭으로 이미지를 업로드한다.
- 지원 형식: JPG, PNG, WEBP
- 최대 파일 크기: 10 MB
- 업로드 전 브라우저에서 이미지 미리보기를 제공한다.

### FR-2. 재료 인식 API 호출
- 업로드된 이미지를 Base64로 인코딩하여 OpenRouter API에 전송한다.
- 사용 모델: `google/gemma-3-27b-it:free`
- 시스템 프롬프트: 이미지 속 식재료만 JSON 배열로 반환하도록 지시
  ```
  You are a food ingredient detector.
  Analyze the image and return ONLY a JSON array of ingredient names in Korean.
  Example: ["계란", "당근", "우유", "치즈"]
  Do not include any other text.
  ```
- 응답은 JSON 파싱 후 재료 목록 배열로 저장한다.

### FR-3. 결과 표시
- 인식된 재료를 태그(pill) 형태로 화면에 표시한다.
- 사용자가 잘못 인식된 재료를 삭제하거나, 재료를 직접 추가할 수 있다.
- "레시피 추천받기" 버튼을 누르면 Step 2로 재료 목록을 전달한다.

## 비기능 요구사항
- API 응답 대기 중 로딩 스피너를 표시한다.
- API 오류(429 Rate Limit 등) 발생 시 사용자에게 재시도 안내 메시지를 표시한다.

## 출력 데이터 형식 (→ Step 2 입력)
```json
{
  "ingredients": ["계란", "당근", "우유", "치즈"],
  "image_url": "data:image/jpeg;base64,..."
}
```
